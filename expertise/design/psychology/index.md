# Directory Purpose

The `psychology` directory explores how human cognition, biases, and emotions impact how users interact with software interfaces.

# Key Concepts

- Reducing mental strain
- Designing forgiving interfaces
- Aligning with user expectations

# File Map

- `cognitive-load.md` — minimizing complexity and decision fatigue
- `error-psychology.md` — writing helpful error messages, preventing mistakes
- `persuasive-design.md` — ethical nudges, scarcity, and social proof
- `user-mental-models.md` — matching UI to how users expect things to work

# Reading Guide

If users find the app confusing → read `cognitive-load.md`
If users make frequent mistakes → read `error-psychology.md`
If trying to increase conversion → read `persuasive-design.md`
