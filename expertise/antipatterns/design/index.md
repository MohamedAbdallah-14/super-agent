# Directory Purpose

The `design` antipatterns directory documents bad practices in User Interface (UI) and User Experience (UX) design, including unethical practices and accessibility failures.

# Key Concepts

- Ethical design vs Dark patterns
- User friction and cognitive overload
- Accessible and inclusive design failures

# File Map

- `accessibility-antipatterns.md` — contrast issues, missing ARIA, bad focus
- `dark-patterns.md` — manipulative UX, forced continuity, hidden costs
- `ui-antipatterns.md` — cluttered interfaces, inconsistent styling
- `ux-antipatterns.md` — confusing navigation, friction in user flows

# Reading Guide

If auditing product ethics → read `dark-patterns.md`
If reviewing interface layout → read `ui-antipatterns.md`
If checking a11y compliance → read `accessibility-antipatterns.md`
