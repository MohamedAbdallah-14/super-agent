# Directory Purpose

The `antipatterns` directory categorizes common mistakes, technical debt, and bad practices across all areas of software engineering. It helps AI agents and developers recognize and avoid solutions that might seem effective but introduce long-term problems.

# Key Concepts

- Identifying and resolving technical debt
- Avoiding structural and architectural mistakes
- Recognizing poor code quality and smells
- Avoiding bad UI/UX and dark patterns
- Preventing common security and scaling pitfalls

# File Map

- `index.md` — semantic map of the antipatterns directory

# Subdirectories

- `/backend` — database, caching, auth, and microservices flaws
- `/code` — general code smells, async issues, and testing mistakes
- `/design` — bad UI/UX, dark patterns, and accessibility fails
- `/frontend` — SPA, CSS, and platform-specific UI antipatterns
- `/performance` — premature optimization and scaling errors
- `/process` — flaws in code review, deployment, and AI coding
- `/security` — security theater, vulnerability patterns, and secrets mismanagement

# Reading Guide

If reviewing general code quality → read `/code`
If evaluating system design → read `/backend` or `/performance`
If auditing security practices → read `/security`
