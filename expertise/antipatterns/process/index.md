# Directory Purpose

The `process` antipatterns directory covers flaws in engineering workflows, collaboration, and deployment pipelines that reduce team velocity and quality.

# Key Concepts

- Review and collaboration bottlenecks
- Unsafe deployment practices
- Mismanagement of technical debt
- Misuse of AI coding tools

# File Map

- `ai-coding-antipatterns.md` — blind trust in AI, missing context, skipped reviews
- `code-review-antipatterns.md` — nitpicking, rubber-stamping, massive PRs
- `deployment-antipatterns.md` — manual deployments, Friday releases, lack of rollback
- `technical-debt-antipatterns.md` — ignoring debt, rewriting from scratch unnecessarily

# Reading Guide

If optimizing CI/CD workflows → read `deployment-antipatterns.md`
If improving team reviews → read `code-review-antipatterns.md`
If using AI for code gen → read `ai-coding-antipatterns.md`
