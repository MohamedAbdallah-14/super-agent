# Governance

SuperAgent is currently maintainer-led.

## Decision model

- maintainers decide the active product direction
- design and implementation changes should cite the active plans and design decisions
- breaking direction changes must update the canonical plans and docs, not just code

## Maintainer responsibilities

- keep active docs truthful
- enforce the host-native product boundary
- require evidence for major claims
- maintain host-native architecture alignment

## Contribution ladder

1. contributor
2. repeat contributor
3. reviewer
4. maintainer

The ladder is based on sustained high-signal contributions, not only code volume.
